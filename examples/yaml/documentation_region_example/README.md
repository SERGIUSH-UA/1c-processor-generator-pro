# Documentation Region Example

Приклад використання регіону `#Область Документация` в файлі `handlers.bsl` для документування модуля форми.

## Особливості

- ✅ Документація в одному файлі з handlers (`handlers.bsl`)
- ✅ Використання регіону `#Область Документация...#КонецОбласті`
- ✅ Безпечне використання прикладів коду з `Функция...КонецФункции` в коментарях
- ✅ Автоматичне витягування та вставка в `Module.bsl`

## Структура

```
documentation_region_example/
├── config.yaml          # YAML конфігурація
├── handlers.bsl         # BSL handlers + регіон Документация
└── README.md            # Цей файл
```

## Регіон Документация

В файлі `handlers.bsl` на початку додається регіон документації:

```bsl
#Область Документация

// Ваша документація тут
// Може бути багато рядків
//
// Приклади коду безпечні:
// Функция ПрикладФункції()
//     Повернення 123;
// КонецФункції

#КонецОбласті

// Тут йдуть звичайні процедури...
Процедура ПриОткрытии(Отказ)
    ...
КонецПроцедуры
```

## Генерація

```bash
python -m 1c_processor_generator yaml \
    --config examples/yaml/documentation_region_example/config.yaml \
    --handlers-file examples/yaml/documentation_region_example/handlers.bsl \
    --output output/ДокументаціяПриклад
```

## Результат

Згенерований `Module.bsl` матиме структуру:

```bsl
#Область Документация

// Вся ваша документація з handlers.bsl

#КонецОбласті

#Область ОбработчикиСобытийФормы
...
#КонецОбласті

...
```

## Переваги

1. **Все в одному файлі** - LLM генерує тільки 2 файли: `config.yaml` + `handlers.bsl`
2. **Безпечні приклади коду** - регіон витягується ПЕРЕД парсингом процедур
3. **Зручність** - не потрібні окремі файли документації
4. **Гнучкість** - можна писати необмежену кількість документації
